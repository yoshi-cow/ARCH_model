## ARCH入門ガイド（修正版）

* **概要**
  ARCH (Autoregressive Conditional Heteroskedasticity) モデルは、時間とともに **分散（ボラティリティ）が変動する** 時系列を扱うためのモデルです。系列の平均構造は単純でも、誤差項の分散が過去のショック（過去の平方誤差）に依存して変化する点が特徴です。

* **基本設定**
  観測系列を
  [
  y_t = \mu + \varepsilon_t
  ]
  とおき、
  [
  \varepsilon_t = \sigma_t z_t
  ]
  とします。ここで (z_t) は **平均 0・分散 1 の独立同分布 (iid)** と仮定し、しばしば (z_t \sim N(0,1))（標準正規）を仮定します。
  このとき
  [
  \sigma_t^2 = \mathrm{Var}(\varepsilon_t \mid \mathcal{F}*{t-1})
  ]
  を **条件付き分散** と呼びます。ARCH(q) モデルでは
  [
  \sigma_t^2 = \omega + \alpha_1 \varepsilon*{t-1}^2 + \cdots + \alpha_q \varepsilon_{t-q}^2
  ]
  と表し、

  * (\omega > 0)
  * (\alpha_i \ge 0)
    を仮定して分散の正値性を担保します。
    このとき、(\sum_{i=1}^q \alpha_i < 1) が満たされていれば、**無条件分散**
    [
    \mathrm{Var}(\varepsilon_t) = \frac{\omega}{1 - \sum_{i=1}^q \alpha_i}
    ]
    が存在し、過程は分散の意味で定常になります。

* **直感**

  * 大きなショック（(\varepsilon_{t-1}^2) が大きい）後には (\sigma_t^2) が増加し、**次期の変動が大きくなりやすい**。
  * ショックが小さい期間には (\sigma_t^2) も小さくなり、**穏やかな変動が続きやすい**。
  * このように「大きな変動の後に大きな変動が続く」という **分散（ボラティリティ）のクラスタリング** を表現できます。

* **推定手法**

  * 最尤法（Maximum Likelihood Estimation）が一般的で、(z_t) に正規分布を仮定すると尤度関数が明示的に書けます（実務では t 分布などを使うこともあります）。
  * 分散パラメータは非線形最適化で推定され、(\omega) と (\alpha_i) には非負の制約を課します（実装ではパラメータ変換で自動的に制約を満たすようにすることが多い）。

* **適用ステップ**

  1. まず系列の **平均構造をモデル化・除去** し（トレンド除去、差分、AR モデルなど）、残差系列 (\hat{\varepsilon}_t) を作る。
  2. 残差に自己相関がほとんど残っていないことを確認する（平均構造が十分に説明されているかのチェック）。
  3. 残差平方 (\hat{\varepsilon}_t^2) に対して Engle の **ARCH-LM 検定** を行い、ARCH 効果の有無を判定する。
  4. ARCH の次数 (q) は AIC/BIC や残差診断（標準化残差の自己相関など）に基づいて選択する。

* **評価と診断**

  * 推定された条件付き分散 (\hat{\sigma}_t^2) の推移をプロットし、実際の系列の変動パターンと整合的か確認する。
  * **標準化残差** (\hat{z}_t = \hat{\varepsilon}_t / \hat{\sigma}_t) と、その**平方** (\hat{z}_t^2) について

    * 自己相関関数（ACF）
    * Ljung-Box 検定
      などを用いて、**平均構造・分散構造が十分に捉えられているか** をチェックする。
      特に (\hat{z}_t^2) に有意な自己相関が残っていないかが、ARCH/GARCH モデルの診断では重要。

* **限界と拡張**

  * ARCH は次数 (q) が大きくなるとパラメータ数が増え、推定が煩雑になります。
  * より持続的なボラティリティの動きを表現するため、GARCH などの拡張モデルが広く用いられます。
  * 実データでは裾の分布が厚くなることが多いため、誤差分布として正規分布ではなく **Student-t 分布などを用いる** こともあります（fat tail 対応）。

## GARCHモデルへの拡張（修正版）

* **ARCHモデルの主な弱点**

  * 分散が過去の「平方誤差」にのみ依存しており、**ボラティリティの長期的な持続性（persistence）** を表現しようとすると、高次の ARCH(q) が必要になる。
  * (q) を大きくすると **パラメータ数が増加して推定が不安定・非効率** になりやすい。
  * 金融時系列などで観察される、非常にゆっくりと減衰するボラティリティの動きには、低次の ARCH では柔軟性が不足することが多い。

* **GARCH(1,1)モデルの基本形**
  ARCH の弱点を補う代表的な拡張が GARCH (Generalized ARCH) モデルです。
  最も基本的な GARCH(1,1) モデルでは、条件付き分散は
  [
  \sigma_t^2 = \omega + \alpha_1 \varepsilon_{t-1}^2 + \beta_1 \sigma_{t-1}^2
  ]
  で表されます。ここで

  * (\omega > 0)
  * (\alpha_1 \ge 0)（直近のショックへの反応の強さ）
  * (\beta_1 \ge 0)（過去の分散の持続性）
    を仮定し、さらに **定常性のために (\alpha_1 + \beta_1 < 1)** を仮定することが一般的です。
    このとき無条件分散は
    [
    \mathrm{Var}(\varepsilon_t) = \frac{\omega}{1 - \alpha_1 - \beta_1}
    ]
    となります。

* **GARCHの直感と ARCH との違い**

  * ARCH では「過去のショック（平方誤差）」だけを見て分散を更新していたのに対し、GARCHでは **過去の分散 (\sigma_{t-1}^2) も直接組み込む** ことで、ボラティリティの持続性を自然に表現できます。
  * (\alpha_1) が大きいと、「新しいショック」に敏感に反応し、(\beta_1) が大きいと、一度高くなったボラティリティが **なかなか下がらない（高いボラティリティが持続しやすい）**。
  * 実務的には、多くの金融データで GARCH(1,1) だけでも十分に良いフィットをすることが多く、ARCH よりも少数のパラメータで **「ゆっくり減衰するボラティリティ」** を表現できます。

* **ARCH との関係**

  * GARCH(p,q) モデルは、(\varepsilon_t^2) に対する **ARMA 型のモデル** とみなすことができます（例えば GARCH(1,1) は (\varepsilon_t^2) に対する ARMA(1,1) に対応）。
  * ARCH(q) モデルは GARCH(p,q) において (p=0) の特殊ケース、すなわち **GARCH(0,q) = ARCH(q)** とみなせます（GARCH(1,1) の場合は (\beta_1 = 0) が ARCH(1) に対応）。
  * したがって、ARCHで表現できる現象は GARCH でも表現でき、そのうえで **より柔軟な分散ダイナミクス** を扱える点が GARCH の利点です。

GARCH の理解を深めるためには、次のステップとして **GARCH(1,1) の条件付き尤度の形・定常条件・長期平均分散（無条件分散）** などを数式レベルで確認していくと良いです。
